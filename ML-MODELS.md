# ML-модели проекта

В проекте CineVibe ML-компоненты расположены в директории `analytics/ml`. В данной папке находятся два основных скрипта машинного обучения:

- `recommender.py` — гибридная рекомендательная модель;
- `predict_rating.py` — NLP-модель для прогнозирования оценки по тексту отзыва.

Данные модели используются для персонализации пользовательского опыта, автоматического анализа отзывов и формирования рекомендаций контента.

## Гибридная рекомендательная модель `recommender.py`

Файл `recommender.py` реализует гибридную рекомендательную систему, предназначенную для подбора фильмов, сериалов и игр на основе пользовательских оценок и характеристик контента.

Модель использует два подхода:

- content-based filtering;
- collaborative filtering.

Объединение этих подходов позволяет учитывать как свойства самого контента, так и поведение пользователей.

### Content-Based Filtering

Content-based filtering формирует рекомендации на основе признаков контента. Для каждого объекта из таблицы `content` система получает следующие данные:

- `id`;
- `title`;
- `genre`;
- `emotional_cloud`;
- `perception_map`;
- `hype_index`;
- `avg_rating`.

Жанры контента преобразуются в числовой вид с помощью `MultiLabelBinarizer`. Это позволяет представить набор жанров как бинарный вектор, где `1` означает наличие жанра, а `0` — его отсутствие.

Например:

```text
Жанры: Action, Drama

Вектор:
Action = 1
Drama = 1
Comedy = 0
Horror = 0
Sci-Fi = 0
```

Также используются эмоциональные признаки из поля `emotional_cloud`. В модели учитываются следующие эмоции:

```text
joy, sadness, anger, fear, surprise, disgust,
anticipation, trust, awe, tension, excitement
```

Каждое значение отражает степень выраженности определенной эмоции в контенте.

Дополнительно используется поле `perception_map`, которое описывает восприятие контента по отдельным аспектам:

```text
plot, acting, visuals, soundtrack, originality, pacing, atmosphere
```

Эти признаки также преобразуются в числовой вектор.

После обработки все признаки объединяются в один общий вектор:

```text
content vector = genre features + emotional features + perception features
```

Затем для всех объектов рассчитывается косинусная близость. Она показывает, насколько похожи два объекта между собой по набору признаков. Чем ближе значение к `1`, тем выше сходство.

Таким образом, если пользователь высоко оценил контент с определенными жанрами, эмоциями и особенностями восприятия, система может подобрать похожие объекты.

### Collaborative Filtering

Collaborative filtering использует историю пользовательских оценок. Для этого из таблицы `reviews` выбираются записи, где рейтинг не является пустым:

```sql
SELECT user_id, content_id, rating
FROM reviews
WHERE rating IS NOT NULL
```

На основе этих данных строится матрица "пользователь-контент". В строках находятся пользователи, в столбцах — объекты контента, а значения представляют собой оценки пользователей.

Пример:

```text
             Film A   Film B   Film C
User 1          10       8        0
User 2           9       0        7
User 3           0       8        6
```

Значение `0` означает, что пользователь еще не оценивал данный объект.

После построения матрицы она транспонируется, чтобы сравнивать объекты контента между собой. Далее рассчитывается косинусная близость между объектами. Если два фильма, сериала или игры получают похожие оценки от одних и тех же пользователей, они считаются похожими.

Например, если пользователи, которым понравился фильм A, также высоко оценивают фильм B, то система считает эти два фильма близкими и может рекомендовать фильм B пользователям, которым понравился фильм A.

### Гибридизация подходов

После расчета content-based и collaborative сходства система объединяет их в единую гибридную матрицу сходства.

В проекте используется следующая формула:

```text
hybrid_similarity = 0.7 * collaborative_similarity + 0.3 * content_similarity
```

Это означает, что при наличии пользовательских оценок 70% итогового сходства формируется на основе поведения пользователей, а 30% — на основе характеристик контента.

Если отзывов и оценок в системе еще нет, collaborative filtering не может быть построен. В этом случае модель использует только content-based filtering.

Такой подход позволяет системе работать как при наличии большого количества пользовательских данных, так и в ситуации, когда данных еще мало.

### Генерация рекомендаций

После построения гибридной матрицы сходства модель формирует рекомендации для каждого пользователя.

Алгоритм генерации рекомендаций:

1. Получить список всех пользователей, оставивших оценки.
2. Для каждого пользователя определить контент, который он уже оценил.
3. Исключить уже оцененные объекты из списка кандидатов.
4. Для каждого оставшегося объекта рассчитать прогнозный score.
5. Score рассчитывается на основе сходства кандидата с уже оцененными объектами.
6. Отобрать топ-20 объектов с наибольшим score.
7. Сохранить рекомендации в таблицу `recommendations`.

Прогнозный score рассчитывается как средневзвешенная оценка:

```text
predicted_score = weighted_sum / similarity_sum
```

Где:

```text
weighted_sum — сумма произведений сходства на пользовательскую оценку;
similarity_sum — сумма значений сходства.
```

Если кандидат похож на несколько объектов, которые пользователь оценил высоко, итоговый score будет высоким.

Результаты сохраняются в базу данных в таблицу `recommendations` со следующими полями:

```text
user_id
content_id
score
reason
```

Поле `reason` содержит объяснение причины рекомендации.

### Статус выполнения модели

Во время работы модель обновляет статус выполнения в таблице `ml_task_logs`. Это позволяет backend-части отслеживать прогресс выполнения ML-задачи.

В статус записываются:

```text
task_name
status
progress
message
updated_at
```

Например, система может показать, что модель находится на этапе загрузки данных, построения content-based модели, построения collaborative filtering или сохранения результатов.

## NLP-модель прогнозирования рейтинга `predict_rating.py`

Файл `predict_rating.py` реализует модель анализа тональности текста для автоматического прогнозирования рейтинга по содержанию отзыва.

Данная модель используется в случаях, когда пользователь оставил текстовый отзыв, но не указал числовую оценку. Система анализирует текст и определяет предполагаемый рейтинг.

### Используемая модель

В проекте используется предобученная NLP-модель:

```text
nlptown/bert-base-multilingual-uncased-sentiment
```

Модель подключается через библиотеку `transformers` с помощью pipeline:

```python
pipeline("sentiment-analysis", model="nlptown/bert-base-multilingual-uncased-sentiment")
```

Данная модель основана на архитектуре BERT и предназначена для многоязычного анализа тональности текста. Она способна работать с отзывами на разных языках и определять эмоциональную окраску текста.

### Назначение модели

Основная задача модели — преобразовать текстовый отзыв в числовую оценку. Это позволяет использовать даже те отзывы, где пользователь не указал рейтинг вручную.

Например, отзыв:

```text
Фильм получился очень атмосферным, актеры сыграли отлично, сюжет держит в напряжении до конца.
```

может быть определен моделью как положительный и получить высокую оценку.

Отзыв:

```text
Сюжет скучный, персонажи не раскрыты, фильм слишком затянут.
```

может быть определен как отрицательный и получить низкую оценку.

### Принцип работы

Сначала скрипт получает из базы данных отзывы без рейтинга:

```sql
SELECT id, content_id, user_id, content
FROM reviews
WHERE rating IS NULL
  AND content IS NOT NULL
  AND content != '';
```

Далее каждый текст передается в NLP-модель. Перед обработкой текст ограничивается первыми 512 символами, так как BERT-модели имеют ограничение на длину входной последовательности.

После анализа модель возвращает результат в формате:

```text
1 star
2 stars
3 stars
4 stars
5 stars
```

Каждое значение соответствует уровню тональности текста:

```text
1 star  — резко отрицательный отзыв;
2 stars — скорее отрицательный отзыв;
3 stars — нейтральный или смешанный отзыв;
4 stars — скорее положительный отзыв;
5 stars — положительный отзыв.
```

### Преобразование результата в шкалу проекта

Так как в проекте используется шкала рейтинга от 1 до 10, результат NLP-модели преобразуется из 5-звездочной шкалы в 10-балльную.

Преобразование выполняется по формуле:

```text
rating = stars * 2
```

Соответствие шкал:

```text
1 star  -> 2/10
2 stars -> 4/10
3 stars -> 6/10
4 stars -> 8/10
5 stars -> 10/10
```

После преобразования рейтинг записывается в базу данных для соответствующего отзыва.

### Сохранение результата

После анализа всех отзывов скрипт обновляет таблицу `reviews`. Для каждого отзыва, у которого отсутствовала оценка, устанавливается предсказанный рейтинг.

Таким образом, текстовые отзывы без числовой оценки становятся пригодными для дальнейшей аналитики и могут использоваться рекомендательной системой.

### Статус выполнения модели

Как и рекомендательная модель, `predict_rating.py` обновляет статус выполнения в таблице `ml_task_logs`.

Во время выполнения фиксируются этапы:

```text
начало процесса прогнозирования;
загрузка NLP-модели;
анализ отзывов;
сохранение результатов в базу данных;
завершение процесса.
```

Это позволяет отслеживать выполнение ML-задачи через backend и отображать состояние процесса в интерфейсе или административной панели.

## Используемые библиотеки

Для реализации ML-компонентов используются следующие библиотеки:

```text
pandas — загрузка и обработка табличных данных;
numpy — работа с числовыми массивами и векторами;
scikit-learn — кодирование признаков, нормализация и расчет косинусной близости;
sqlalchemy — подключение к базе данных;
pymysql — драйвер для работы с MySQL;
python-dotenv — загрузка переменных окружения;
transformers — использование предобученной NLP-модели;
torch — backend для работы модели BERT.
```

## Итог

В проекте реализованы две основные ML-модели.

Первая модель, `recommender.py`, отвечает за формирование персонализированных рекомендаций. Она использует гибридный подход, объединяя анализ характеристик контента и поведение пользователей.

Вторая модель, `predict_rating.py`, отвечает за автоматическое определение рейтинга по тексту отзыва. Она использует предобученную многоязычную BERT-модель для анализа тональности текста.

Совместно эти модели позволяют системе CineVibe не только рекомендовать пользователям наиболее подходящий контент, но и автоматически обрабатывать текстовые отзывы, повышая качество аналитики и персонализации.
